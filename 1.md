# 🔐 SSH (Secure Shell) 核心知识点与考点全解析

## 第一层：SSH是什么？（基础概念层）

### 1. 协议定义

|考点|内容|
|---|---|
|**全称**|Secure Shell（安全外壳协议）|
|**本质**|一种**加密的网络传输协议**|
|**核心作用**|为远程登录会话和其他网络服务提供**安全性**|

### 2. 为什么需要SSH？（对比理解）

```
┌─────────────────────────────────────────────────────────┐
│  传统协议（不安全）          │  SSH协议（安全）          │
├─────────────────────────────────────────────────────────┤
│  Telnet - 明文传输密码       │  SSH - 加密传输           │
│  FTP - 明文传输文件          │  SCP/SFTP - 加密传输      │
│  HTTP - 明文传输网页         │  HTTPS - 加密传输         │
└─────────────────────────────────────────────────────────┘
```

**考题陷阱**：题目常把SSH与FTP、HTTP、ICMP放在一起让你选择哪个是"提供安全性的协议"，答案是SSH。

## 第二层：

### 1. OpenSSH软件套件

SSH是一个协议标准，而**OpenSSH**是Linux下最常用的SSH协议实现软件。

|组件|作用|考点|
|---|---|---|
|**sshd**|SSH服务端守护进程|需要启动sshd服务才能接受远程连接|
|**ssh**|SSH客户端命令|用于远程登录|
|**scp**|安全复制命令|基于SSH的文件传输|
|**sftp**|安全文件传输|类似FTP但加密|
|**ssh-keygen**|密钥生成工具|生成公私钥对|

### 2. 默认端口号（高频考点！）

```
┌────────────────────────────────────────┐
│  协议      │  默认端口                  │
├────────────────────────────────────────┤
│  SSH       │  22 ★★★                  │
│  HTTP      │  80                       │
│  FTP       │  21（控制）/ 20（数据）    │
│  Telnet    │  23                       │
│  DNS       │  53                       │
└────────────────────────────────────────┘
```

### 1. 服务启动与配置

|操作|命令|说明|
|---|---|---|

## 第四层：SSH远程登录（应用层）

### 1. 基本登录命令

```bash
# 语法格式
ssh 用户名@服务器IP地址

# 示例
```

## 第五层：SSH文件传输（扩展层）

### 1. SCP命令（Secure Copy）

**核心考点：SCP是基于SSH的安全文件传输命令**

|操作|命令格式|示例|
|---|---|---|
|**上传文件**|`scp 本地文件 用户@IP:远程路径`|`scp test.c user1@10.0.0.50:/tmp/`|
|**下载文件**|`scp 用户@IP:远程文件 本地路径`|`scp root@10.1.0.2:/opt/www/ /opt/`|
|**上传目录**|`scp -r 本地目录 用户@IP:远程路径`|`scp -r /opt/www/ root@10.1.0.2:/opt/`|
|**下载目录**|`scp -r 用户@IP:远程目录 本地路径`|`scp -r root@10.1.0.2:/opt/www/ /opt/`|

**重要参数**：

- `-r`：递归传输目录（传输目录时必须加！）

### 2. SCP vs FTP 对比

|特性|SCP|FTP|
|---|---|---|
|安全性|加密传输|明文传输（不安全）|
|依赖协议|SSH|FTP协议|
|默认端口|22|21|
|使用场景|安全要求高的场景|内网/传统场景|

## 第六层：SSH免密登录（进阶层）⭐⭐⭐

这是考试的重点难点，务必掌握！

### 1. 免密登录原理（公私钥认证）

```
┌─────────────────────────────────────────────────────────────────┐
│                    SSH免密登录原理图                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌──────────┐                              ┌──────────┐       │
│   │  客户端  │                              │  服务端  │       │
│   │ (server1)│                              │ (server2)│       │
│   ├──────────┤                              ├──────────┤       │
│   │          │  1. 生成密钥对               │          │       │
│   │ 私钥     │ ←── ssh-keygen ──→          │          │       │
│   │ (保密!)  │     ┌─────────┐             │          │       │
│   │          │     │公钥     │             │          │       │
│   │          │     │(可公开) │             │          │       │
│   │          │     └─────────┘             │          │       │
│   │          │                              │          │       │
│   │          │  2. 复制公钥到服务端         │          │       │
│   │          │ ────── scp ──────────────→   │          │       │
│   │          │                              │ 公钥存入 │       │
│   │          │                              │authorized│       │
│   │          │  3. 免密登录                 │ _keys文件│       │
│   │          │ ────── ssh ──────────────→   │          │       │
│   │          │ ←───── 验证通过！ ────────   │          │       │
│   └──────────┘                              └──────────┘       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 免密登录完整操作步骤

#### 步骤1：生成密钥对

```bash
ssh-keygen
```

- 会在 `~/.ssh/` 目录下生成两个文件：
    - `id_rsa`：私钥（必须保密！）
    - `id_rsa.pub`：公钥（可以公开）

#### 步骤2：密钥存储位置

|文件|路径|说明|
|---|---|---|
|私钥|`~/.ssh/id_rsa`|客户端保存，绝不外泄|
|公钥|`~/.ssh/id_rsa.pub`|复制到服务端|
|授权文件|`~/.ssh/authorized_keys`|服务端保存所有允许的公钥|

#### 步骤3：复制公钥到服务端

```bash
# 方法1：手动复制
scp ~/.ssh/id_rsa.pub user2@server2:~/.ssh/

# 方法2：使用ssh-copy-id（更简单）
ssh-copy-id user2@server2
```

#### 步骤4：服务端配置

```bash
# 登录服务端后，将公钥追加到authorized_keys
cat id_rsa.pub >> ~/.ssh/authorized_keys
```

### 3. 关键文件总结（必背！）

|文件名|位置|作用|
|---|---|---|
|**id_rsa**|客户端 ~/.ssh/|私钥文件|
|**id_rsa.pub**|客户端 ~/.ssh/|公钥文件|
|**authorized_keys**|服务端 ~/.ssh/|存储所有授权的公钥|
|**known_hosts**|客户端 ~/.ssh/|存储已连接过的主机指纹|

**考题示例**：

> 用于存储所有授权的远程客户系统的公钥，使远程客户端系统能够ssh免密登录的文件是？
> 
> - **A. authorized_keys ✅**
> - B. ssh_keys ❌
> - C. authorized_hosts ❌
> - D. know_hosts ❌

### 综合应用题精选

**题目1**：用户用scp命令把文件上传至IP地址为10.0.0.50的服务器的/tmp目录下，用户名user1

**答案**：

```bash
scp /tmp/target/test.c user1@10.0.0.50:/tmp/
```

**题目2**：使用ssh以user1身份登录服务器

**答案**：

```bash
ssh user1@10.0.0.50
```

**题目3**：配置在server1上以用户名user2免密登录到server2

**答案**：

```bash
# 步骤1：生成密钥
ssh-keygen

# 步骤2：进入密钥目录
cd ~/.ssh

# 步骤3：复制公钥
scp id_rsa.pub user2@server2:/home/user2/.ssh/

# 步骤4：服务端追加到授权文件
cat id_rsa.pub >> authorized_keys
```
